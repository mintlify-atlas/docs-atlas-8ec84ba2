## WWScreenView

A root screen container that handles scrolling, keyboard interactions, and safe area insets. Use this as the main container for all screens in your app.

### Props

<ParamField path="scrollable" type="boolean" default="true">
  When true, wraps content in a ScrollView. Set to false for static screens
</ParamField>

<ParamField path="children" type="React.ReactNode" required>
  Content to display within the screen
</ParamField>

<ParamField path="style" type="StyleProp<ViewStyle>">
  Custom styles applied to the content container
</ParamField>

### Type Definition

```tsx
type Props = PropsWithChildren<ViewProps | ScrollViewProps> & {
  scrollable?: boolean
}
```

### Features

- **Automatic Safe Area Handling**: Applies safe area insets for top and bottom edges
- **Keyboard Dismissal**: Tap anywhere to dismiss the keyboard
- **Theme Integration**: Uses `appPadding` from theme for consistent spacing
- **ScrollView Support**: Configurable scrolling with `keyboardShouldPersistTaps="handled"`

## WWScrollView

A flexible scroll container with gesture handler support. Use this for scrollable regions within a screen.

### Props

<ParamField path="children" type="React.ReactNode" required>
  Content to display within the scroll view
</ParamField>

<ParamField path="containerStyle" type="StyleProp<ViewStyle>">
  Custom styles for the inner content container
</ParamField>

<ParamField path="style" type="StyleProp<ViewStyle>">
  Custom styles for the ScrollView itself
</ParamField>

<ParamField path="horizontal" type="boolean" default="false">
  Enables horizontal scrolling
</ParamField>

<ParamField path="showsVerticalScrollIndicator" type="boolean" default="true">
  Shows/hides the vertical scroll indicator
</ParamField>

<ParamField path="showsHorizontalScrollIndicator" type="boolean" default="true">
  Shows/hides the horizontal scroll indicator
</ParamField>

### Type Definition

```tsx
type Props = ScrollViewProps & {
  containerStyle?: StyleProp<ViewStyle>
}
```

### Features

- **Gesture Handler Integration**: Uses `react-native-gesture-handler` for smooth scrolling
- **Ref Support**: Forwards refs for imperative scroll control
- **Responder Support**: Inner view handles touch responders for nested interactions

## Implementation Details

### Source Code Locations

- **WWScreenView**: `/workspace/source/src/components/ui/WWScreenView.tsx:18-53`
- **WWScrollView**: `/workspace/source/src/components/ui/WWScrollView.tsx:15-25`

### WWScreenView Behavior

When `scrollable={true}` (default):
- Wraps content in `ScrollView`
- Applies `keyboardShouldPersistTaps="handled"` for better form UX
- Sets `minHeight: 100%` on content container
- Adds bottom padding based on safe area insets

When `scrollable={false}`:
- Wraps content in `View`
- Uses `flex: 1` for full-screen layout
- Still applies safe area padding

### Keyboard Handling

Both components work together for optimal keyboard behavior:

```tsx
import { WWScreenView } from '@/components/ui'
import { WWTextInput, WWButton } from '@/components/ui'

// Keyboard automatically dismisses on tap outside
// ScrollView adjusts to keep focused input visible
export const FormScreen = () => {
  return (
    <WWScreenView>
      <WWTextInput label="Field 1" />
      <WWTextInput label="Field 2" />
      <WWTextInput label="Field 3" />
      <WWButton>Submit</WWButton>
    </WWScreenView>
  )
}
```

### Performance Considerations

- Use `scrollable={false}` for screens with maps or heavy graphics
- Use `WWScrollView` for horizontal lists instead of `FlatList` when the number of items is small
- Consider `FlatList` or `FlashList` for large vertical lists instead of `WWScrollView`
