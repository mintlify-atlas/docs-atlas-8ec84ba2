This guide will help you get the Wildlife Watcher mobile app up and running on your development machine.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 18+** installed on your machine
- **Ruby 2.6.10+** for iOS development
- **React Native environment** set up per the [official documentation](https://reactnative.dev/docs/set-up-your-environment)
- **Xcode** (for iOS development) or **Android Studio** (for Android development)

<Note>
  If you haven't set up your React Native development environment yet, follow the [installation guide](/installation) first.
</Note>

## Running on iOS

<Steps>
  <Step title="Start Metro Bundler">
    Open a terminal in the project directory and start Metro:
    ```bash
    npm start
    ```
    Metro is the JavaScript bundler for React Native. Keep this terminal window open.
  </Step>

  <Step title="Run on iOS Simulator">
    In a new terminal window, run:
    ```bash
    npm run ios
    ```
    
    This will build the iOS app and launch it in the iOS Simulator. The first build may take several minutes.
    
    To run on a specific simulator:
    ```bash
    npx react-native run-ios --simulator="iPhone 15 Pro"
    ```
  </Step>

  <Step title="Verify the App">
    Once the app launches, you should see the Wildlife Watcher login screen. The app is now running in development mode.
  </Step>
</Steps>

## Running on Android

<Steps>
  <Step title="Start an Android Emulator">
    Launch an Android emulator from Android Studio, or connect a physical Android device via USB with USB debugging enabled.
  </Step>

  <Step title="Start Metro Bundler">
    In a terminal, start Metro:
    ```bash
    npm start
    ```
  </Step>

  <Step title="Run on Android">
    In a new terminal window, run:
    ```bash
    npm run android
    ```
    
    This will build the Android app and install it on your emulator or connected device.
  </Step>
</Steps>

## Testing on Physical Devices

### iOS Device

1. Connect your iOS device via USB
2. Open the project in Xcode: `ios/WildlifeWatcher.xcworkspace`
3. Select your device from the device dropdown
4. Click the "Run" button or press `Cmd + R`

<Warning>
  You'll need to configure code signing in Xcode with your Apple Developer account.
</Warning>

### Android Device

1. Enable **Developer Options** and **USB Debugging** on your Android device
2. Connect your device via USB
3. Run `adb devices` to verify the device is connected
4. Run `npm run android`

## Understanding the App Structure

Once the app is running, you'll see:

### Main Navigation

The app uses a bottom tab navigator with four main sections:

```typescript src/navigation/BottomTabs.tsx
<Tab.Screen
  name="Devices"
  component={Devices}
  options={{
    tabBarIcon: ({ color }) => <WWIcon source="cellphone-link" size={24} color={color} />,
  }}
/>
<Tab.Screen
  name="Deployments"
  component={Deployments}
  options={{
    tabBarIcon: ({ color }) => <WWIcon source="map-marker" size={24} color={color} />,
  }}
/>
<Tab.Screen
  name="Projects"
  component={Projects}
  options={{
    tabBarIcon: ({ color }) => <WWIcon source="folder" size={24} color={color} />,
  }}
/>
<Tab.Screen
  name="Profile"
  component={Profile}
  options={{
    tabBarIcon: ({ color }) => <WWIcon source="account" size={24} color={color} />,
  }}
/>
```

### Key Screens

- **Devices**: Scan and connect to Wildlife Watcher cameras via Bluetooth
- **Deployments**: View and manage camera deployments on a map
- **Projects**: Organize deployments into projects
- **Profile**: User settings and app configuration

## Developer Tools

### Reload the App

- **iOS**: Press `Cmd + R` in the simulator
- **Android**: Double-tap `R` on your keyboard or shake the device

### Open Developer Menu

- **iOS**: Press `Cmd + D` in the simulator
- **Android**: Press `Cmd + M` (Mac) or `Ctrl + M` (Windows/Linux), or shake the device

### Hot Reloading

React Native supports Fast Refresh, which automatically reloads the app when you save changes to your code. This is enabled by default.

## Common Issues

<AccordionGroup>
  <Accordion title="Metro bundler port already in use">
    If port 8081 is already in use, kill the process:
    ```bash
    npx react-native start --reset-cache
    ```
  </Accordion>

  <Accordion title="iOS build fails">
    Try cleaning the build:
    ```bash
    cd ios
    rm -rf Pods Podfile.lock
    bundle exec pod install
    cd ..
    npm run ios
    ```
  </Accordion>

  <Accordion title="Android build fails">
    Clean the Android build:
    ```bash
    cd android
    ./gradlew clean
    cd ..
    npm run android
    ```
  </Accordion>

  <Accordion title="Bluetooth permissions not working">
    Ensure you've granted the necessary permissions:
    - **iOS**: Location permissions are required for BLE scanning
    - **Android**: Location and Bluetooth permissions must be granted
    
    See the [permissions guide](/guides/permissions) for detailed setup.
  </Accordion>
</AccordionGroup>
